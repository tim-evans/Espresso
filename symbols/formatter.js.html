  6       Advanced String Formatting borrowed from the eponymous Python PEP.
  7 
  8       The formatter follows the rules of Python [PEP 3101][pep]
  9       (Advanced String Formatting) and following most of the
 10       ECMAScript Harmony strawman specification for string formatting
 11       (found [here][strawman]), with the major ommision being the
 12       removal of the bracket `[]` notation for reasons of
 13       consitency and performance.
 14 
 15       To use literal object notation, just pass in one argument for
 16       the formatter. This is optional however, as you can always
 17       absolutely name the arguments via the number in the argument
 18       list. This means that:
 19 
 20           alert(Espresso.format("Hello, {name}!", { name: "world" }));
 21 
 22       is equivalent to:
 23 
 24           alert(Espresso.format("Hello, {0.name}!", { name: "world" }));
 25 
 26       For more than one argument you must provide the position of your
 27       argument.
 28 
 29           alert(Espresso.format("{0}, {1}!", "hello", "world"));
 30 
 31       If your arguments and formatter are "as is"- that is, in order,
 32       and flat objects as you intend them to be, you can write your
 33       template string like so:
 34 
 35           alert(Espresso.format("{}, {}!", "hello", "world"));
 36 
 37       To use the literals `{` and `}`, simply double them, like the following:
 38 
 39           alert(Espresso.format("{lang} uses the {{variable}} format too!", {
 40              lang: "Python", variable: "(not used)"
 41           }));
 42           // => "Python uses the {variable} format too!"
 43 
 44       Check out the examples given for some ideas on how to use it.
 45 
 46       The formatting API uses the special `toFormat` function on an
 47       object to handle the interpretation of the format specifiers.
 48 
 49       The default `toFormat` handler is on `Object.prototype`.
 50 
 51       For an example of a specialized format schema, consider the
 52       following example:
 53 
 54           Localizer = mix({
 55             toFormat: function (spec) {
 56               return this[spec];
 57             }
 58           }).into({});
 59 
 60           _hello = mix(Localizer).into({
 61             en: 'hello',
 62             fr: 'bonjour',
 63             ja: 'こんにちは'
 64           });
 65 
 66           alert(Espresso.format("{:en}", _hello));

 68

 70           // => "bonjour"
 71 
 72           alert(Espresso.format("{:ja}", _hello));

 97     /**
 98       The specifier regular expression.
 99 
100       The groups are:
101 
102         `[[fill]align][sign][#][0][minimumwidth][.precision][type]`
103 
104       The brackets (`[]`) indicates an optional element.
105 
106       The `fill` is the character to fill the rest of the minimum width
107       of the string.
108 
109       The `align` is one of:
110 
111         - `^` Forces the field to be centered within the available space.
112         - `<` Forces the field to be left-aligned within the available
113               space. This is the default.
114         - `>` Forces the field to be right-aligned within the available space.
115         - `=` Forces the padding to be placed after the sign (if any)
116               but before the digits. This alignment option is only valid
117               for numeric types.
118 
119       Unless the minimum field width is defined, the field width
120       will always be the same size as the data to fill it, so that
121       the alignment option has no meaning in this case.
122 
123       The `sign` is only valid for numeric types, and can be one of
124       the following:
125 
126         - `+` Indicates that a sign shoulb be used for both positive
127               as well as negative numbers.
128         - `-` Indicates that a sign shoulb be used only for as negative
129               numbers. This is the default.
130         - ` ` Indicates that a leading space should be used on positive
131               numbers.
132 
133       If the `#` character is present, integers use the 'alternate form'
134       for formatting. This means that binary, octal, and hexadecimal
135       output will be prefixed with '0b', '0o', and '0x', respectively.
136 
137       `width` is a decimal integer defining the minimum field width. If
138       not specified, then the field width will be determined by the
139       content.
140 
141       If the width field is preceded by a zero (`0`) character, this enables
142       zero-padding. This is equivalent to an alignment type of `=` and a
143       fill character of `0`.
144 
145       The 'precision' is a decimal number indicating how many digits
146       should be displayed after the decimal point in a floating point
147       conversion. For non-numeric types the field indicates the maximum
148       field size- in other words, how many characters will be used from
149       the field content. The precision is ignored for integer conversions.
150 
151       Finally, the 'type' determines how the data should be presented.
152 
153       The available integer presentation types are:
154 
155         - `b` Binary. Outputs the number in base 2.
156         - `c` Character. Converts the integer to the corresponding
157               Unicode character before printing.
158         - `d` Decimal Integer. Outputs the number in base 10.
159         - `o` Octal format. Outputs the number in base 8.
160         - `x` Hex format. Outputs the number in base 16, using lower-
161               case letters for the digits above 9.
162         - `X` Hex format. Outputs the number in base 16, using upper-
163               case letters for the digits above 9.
164         - `n` Number. This is the same as `d`, except that it uses the
165               current locale setting to insert the appropriate
166               number separator characters.
167         - ` ` (None) the same as `d`
168 
169       The available floating point presentation types are:
170 
171         - `e` Exponent notation. Prints the number in scientific
172               notation using the letter `e` to indicate the exponent.
173         - `E` Exponent notation. Same as `e` except it converts the
174               number to uppercase.
175         - `f` Fixed point. Displays the number as a fixed-point
176               number.
177         - `F` Fixed point. Same as `f` except it converts the number
178               to uppercase.
179         - `g` General format. This prints the number as a fixed-point
180               number, unless the number is too large, in which case
181               it switches to `e` exponent notation.
182         - `G` General format. Same as `g` except switches to `E`
183               if the number gets to large.
184         - `n` Number. This is the same as `g`, except that it uses the
185               current locale setting to insert the appropriate
186               number separator characters.
187         - `%` Percentage. Multiplies the number by 100 and displays
188               in fixed (`f`) format, followed by a percent sign.
189         - ` ` (None) similar to `g`, except that it prints at least one
190               digit after the decimal point.
191 
192       @type RegExp
193      */

205   /** @ignore */  // Docs are above
206   function format(template, args) {
207     var prev = '', ch,
208         buffer = [],
209         result, idx = 0,
210         len = template.length;
211 
212     for (; idx < len; idx++) {
213       ch = template.charAt(idx);
214 
215       if (prev === closingBrace) {
216         if (ch !== closingBrace) {
217           throw new Error(unmatchedClosing.format(template.slice(0, idx)));
218 
219         // Double-escaped closing brace.
220         } else {
221           buffer[buffer.length] = closingBrace;
222           prev = '';
223           continue;
224         }
225       }
226 
227       // Begin template parsing
228       if (ch === openingBrace) {
229         result = parseField(template.slice(idx + 1), args);
230         buffer[buffer.length] = result[1];
231         idx += result[0]; // continue after the template.
232 
233       // Normal string processing
234       } else if (ch !== closingBrace) {
235         buffer[buffer.length] = ch;
236       }
237       prev = ch;
238     }
239     return buffer.join('');
240   }

242   /** @ignore
243     Parses the template with the arguments provided,
244     parsing any nested templates.
245 
246     @param {String} template The template string to format.
247     @param {Array} args The arguments to parse the template string.
248     @returns {Array} A tuple with the length it ate up and the formatted template.
249    */
250   function parseField(template, args) {
251     var idx = 0, ch, len = template.length,
252         inSpecifier = false, iBrace = 0;
253     for (; idx < len; idx++) {
254       ch = template.charAt(idx);
255       if (!inSpecifier) {
256         if (ch === specifierSeparator) {
257           inSpecifier = true;
258           continue;
259         }
260 
261         // Double-escaped opening brace
262         if (ch === openingBrace) {
263           if (idx === 0) {
264             return [1, openingBrace];
265           } else {
266             throw new Error(unmatchedOpening.format(template.slice(0, idx)));
267           }
268 
269         // Done formatting.
270         } else if (ch === closingBrace) {
271           return [idx + 1, formatField(template.slice(0, idx), args)];
272         }
273 
274       // Format the template's specifier *after* the whole specifier is found.
275       } else {
276         if (ch === openingBrace) {
277           iBrace++;
278         } else if (ch === closingBrace) {
279           iBrace--;
280         }
281 
282         // Spec is done.
283         if (iBrace === -1) {
284           return [idx + 1, formatField(format(template.slice(0, idx), args), args)];
285         }
286       }
287     }
288     throw new Error(unmatchedOpening.format(template));
289   }

299   function formatField(value, args) {
300     var iSpec = value.indexOf(specifierSeparator),
301         spec, res;
302     iSpec = iSpec === -1 ? value.length : iSpec;
303     spec = value.slice(iSpec + 1);
304     value = value.slice(0, iSpec);
305 
306     // Got `{}`; shift off the first argument passed in.
307     if (value === '') {
308       res = args.shift();
309 
310     // Return the object referenced by the property path given.
311     } else {
312       res = Espresso.getObjectFor(value, args);
313 
314       // Allow for references to object literals
315       if (typeof res === "undefined" &&
316           Array.isArray(args) && args.length === 1 && Espresso.hasValue(args[0])) {
317         res = args[0].get ? args[0].get(value) : Espresso.getObjectFor(value, args[0]);
318       }
319     }
320 
321     if (!spec) {
322       return res;
323     }
324 
325     return res != null && res.toFormat ? res.toFormat(spec) : res;
